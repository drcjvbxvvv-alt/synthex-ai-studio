核心問題

每次對話，歸零的代價

你六個月前踩過 Stripe Webhook 冪等性的坑，修好了，寫了 commit。但下次 Agent 幫你實作退款時，它不知道——以完全一樣的方式再踩一次。這不是 Agent 不夠聰明，而是它沒有記憶。

核心問題與解法
問題
AI 每次對話對專案一無所知，重複踩過同樣的坑
→
解法
在專案中建立可被 Agent 查詢的長期記憶庫
→
結果
Agent 帶著歷史知識寫程式，不再重複錯誤
自動正向閉環
Agent 完成任務，結構化 commit
↓
Brain 提取決策與踩坑
↓
下次 Agent 啟動時即知道
↓
Agent 不重複踩坑，寫更好的 commit

記憶架構

Atkinson-Shiffrin 三層記憶

架構直接對應認知科學的多儲存記憶模型，三層各司其職，get_context() 統一讀取並融合輸出。

L1a
工作記憶
Working Memory
當前任務的即時注意力，對話結束後自動清除。存儲今日筆記與當前任務脈絡。
sessions table
自動 TTL 清除

情節記憶

「那次 JWT 事件是什麼時候？」對應 git commit 歷史、事件序列，支援時光機查詢 temporal_query。

PREVENTS / CAUSES / REQUIRES

知識類型 --kind

L3 語意記憶的五種節點類型，從踩坑到架構決策全面覆蓋。

Pitfall
踩坑記錄
曾經犯過的錯誤、隱藏的陷阱。信心分數讓 Agent 評估可信度。
brain add "Stripe Webhook 重複觸發" --kind Pitfall
Decision
架構決策
為什麼選 A 而不選 B。保留決策背後的脈絡，防止未來重複討論。
brain add "選 PostgreSQL 而非 MongoDB" --kind Decision
Rule
工程規則
必須遵守的技術約定。Brain 回傳後 Agent 將其視為硬性約束。
brain add "JWT 必須使用 RS256" --kind Rule
ADR
架構決策記錄
正式的架構決策文件，對應 Architecture Decision Record 標準。
brain add "ADR-001 採用事件驅動架構" --kind ADR
Note
一般筆記
其他值得記住但不屬於上述類型的資訊。
brain add "新人 onboarding 注意事項" --kind Note

工程哲學

五個設計原則

對抗遺忘，而非追求全知。記憶是工程團隊的基礎設施。

01
零依賴優先
核心功能只依賴 Python 標準函式庫與 SQLite。不需要 Docker、Redis、向量資料庫，brain setup 一行指令，無需配置任何外部服務。
02
單一文件，備份即複製
所有記憶存在 .brain/brain.db，一個標準 SQLite 文件。備份只需 cp brain.db brain.db.bak，不需要備份腳本或管理員介入。
03
降級永遠可用
Brain 的故障模式是「靜默降級」，而不是「阻斷流程」。無 API Key 時跳過 LLM 提取，無向量索引時降級為 FTS5 搜尋，知識庫為空時回傳空字串。
04
信心分級，而非二元判斷
知識不是「對」或「錯」，而是有不同程度的可信度。六因子動態衰減公式讓信心隨時間、版本差距、程式碼活動即時更新。
05
人機協作，而非全自動
自動化降低摩擦，但不替代判斷。brain sync（git hook）自動記錄每次 commit，提供原始素材。真正有價值的知識，仍然需要工程師主動用 brain add 確認和補充。這不是設計缺陷，是刻意的選擇——全自動提取的知識品質不足以直接信任。
降級對照表
無 API Key
跳過 LLM 提取，仍記錄 commit 基本資訊，confidence 0.2
無向量索引
降級為 FTS5 關鍵字搜尋，語意匹配能力降低但仍可用
MCP 連線失敗
CLAUDE.md 提示 Agent 主動查詢，降級為手動模式
知識庫為空
回傳空字串，不阻擋 Agent 正常運作

快速開始

安裝 × 初始化

三步驟完成：安裝套件、初始化專案、連接 Agent。系統需求：Python 3.10+，無需外部服務。

安裝選項
# 標準安裝（含向量語意搜尋）
$ pip install project-brain
 
# 含 MCP Server（Claude Code / Cursor）
$ pip install "project-brain[mcp]"
 
# 含 Anthropic SDK（AI 知識提取）
$ pip install "project-brain[anthropic]"
 
# 完整安裝
$ pip install "project-brain[all]"
 
# 初始化專案
$ cd /your/project && brain setup
 
# 驗證 sqlite-vec
$ brain doctor
三步驟快速開始
01
加入第一條知識
brain add "JWT 必須使用 RS256" \
  --kind Rule --confidence 0.9
02
驗證可以查到
brain ask "JWT 設定"
brain ask "支付退款"
03
連接 Claude Code
加入 MCP 設定後，Agent 在每次任務開始時自動呼叫 get_context() 取得相關知識。
已知限制與設計邊界
~75%
語意召回率：FTS5 對語意相近但用詞不同的查詢效果有限。Phase 1 將加入向量語意搜尋。
重複
三層輸出可能重複：L2 episode 與 L3 node 可能說同一件事。Phase 4 計劃自動建立 DERIVES_FROM 邊。
手動
scope 需手動指定：忘記加 --scope 知識會變成全域污染。Phase 5 計劃從目錄自動推導。
邊界
無法捕捉隱性知識：只能記錄已被意識到的決策。這是設計邊界，不修。
