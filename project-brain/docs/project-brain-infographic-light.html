Engineering Memory Infrastructure

Project Brain

為 AI Agent 設計的工程記憶基礎設施 —
讓每次對話都能承接上一次的決策、規則與踩坑。

v0.1.0
Python 3.10+
MCP Compatible
Zero External Deps
核心問題與解法
問題
AI 每次對話對專案一無所知，重複踩過同樣的坑
→
解法
在專案中建立可被 Agent 查詢的長期記憶庫
→
結果
Agent 帶著歷史知識寫程式，不再重複錯誤
自動閉環
Agent 完成任務，結構化 commit
↓
Brain 提取決策與踩坑
↓
下次 Agent 啟動時即知道
↓
Agent 不重複踩坑，寫更好的 commit
三層記憶架構（Atkinson-Shiffrin）
L1a
工作記憶 Working Memory
當前任務脈絡，對話結束後自動清除。今日筆記、即時注意力。
L2
情節記憶 Episodic Memory
git commit 歷史、事件序列。支援 temporal_query 時光機查詢。
L3
語意記憶 Semantic Memory
精煉自 L2 的長期知識。支援因果關係圖譜：PREVENTS / CAUSES / REQUIRES。
get_context() 統一讀取三層，融合輸出
知識類型
Pitfall 踩坑記錄，防止重蹈覆轍
Decision 架構決策，選 A 而不選 B 的理由
Rule 工程規則，必須遵守的技術約定
ADR 正式架構決策記錄文件
Note 一般筆記，其他值得記住的資訊

六因子知識衰減公式
信心 = F1 × F2 × F3 × F4 × F5 + F7
F1

F3
程式碼活動信號

F4

F7

brain status 記憶庫統計與健康狀態

brain serve 啟動 REST API (port 7891) 或 MCP Server
brain webui D3.js 知識圖譜視覺化 (port 7890)

工程哲學
1
零依賴優先
核心只依賴 Python 標準函式庫 + SQLite，無需 Docker / Redis / 向量資料庫
2
單一文件，備份即複製
所有記憶存在 .brain/brain.db，cp brain.db brain.db.bak 即完成備份
3
降級永遠可用
任何元件失效都只靜默降級，不阻斷工作流
4
信心分級，而非二元判斷
知識不是對或錯，而是有動態信心分數，隨 git 歷史自動更新
5
人機協作，非全自動
git hook 提供原始素材，真正有價值的知識仍需工程師主動確認
