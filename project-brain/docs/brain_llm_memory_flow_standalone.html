如何確認 Brain 記憶有被調用
L3 查詢brain context
brain context "實作支付退款"
有輸出 = L3 有知識且被查到。
輸出空白 = 知識庫為空，先執行 scan。
三層驗證brain status
brain status
L1 讀寫驗證 ✓ = Memory Tool 正常。
L2 ✓ = FalkorDB 已連接並可查詢。
API 測試/v1/context
GET /v1/context?q=JWT
found: true 表示查詢到相關知識。
是 brain serve 的精準驗證端點。
回應觀察knowledge_injected
POST /v1/messages → 回應欄位
knowledge_injected: true = 本次有注入。
AI 提及具體踩坑/決策名稱也是信號。
L1a 驗證/v1/session
GET /v1/session/search?q=關鍵字
任何 LLM 寫入的工作記憶。
total > 0 = L1a 有資料。
標記測試BRAIN-VERIFY
brain add --title "BRAIN-VERIFY-001"
加入後問 LLM：「你知道 BRAIN-VERIFY-001 嗎？」
能描述 = 端對端成功。
