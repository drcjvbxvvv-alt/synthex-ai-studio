## SaaS 財務指標快速計算

當需要評估 SaaS 產品的財務可行性時：

```
LTV（用戶終身價值）= ARPU × 平均留存月數
CAC（用戶獲取成本）= 行銷費用 / 新增用戶數
LTV/CAC 比率 > 3：健康的 SaaS 業務
LTV/CAC 比率 < 1：每獲取一個用戶都在虧錢

損益平衡點 = 固定月成本 / 每用戶月利潤
例：月固定成本 $500，每用戶月利潤 $5 → 需要 100 付費用戶
```

## 常見可行性問題和解法

```
問題：Stripe 手續費吃掉了利潤
解法：確保平均訂單金額 > $20（手續費比例才合理）
     或改用本地支付（台灣：綠界、藍新）

問題：Vercel 免費層流量不夠
解法：靜態資源放 Cloudflare R2（無限流量費用）
     或使用 Cloudflare Pages（更高免費額度）

問題：Neon auto-suspend 造成冷啟動延遲
解法：升級到 Neon 付費方案（disable auto-suspend）
     或改用 Railway PostgreSQL（長駐不暫停）

問題：Serverless 連線池超限
解法：加入 PgBouncer 或 Prisma Accelerate
     連線字串加 ?connection_limit=3
```
